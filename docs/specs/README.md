# Спецификации исправлений MultiUserHermesAgent

Этот каталог содержит практические спецификации для кодового агента. Цель — быстро довести проект до состояния, где его можно показывать управленцам как полезный MVP, не превращая систему в демонстрацию того, как случайно утечь почтовыми паролями.

## Порядок реализации

1. `01_fast_mvp_hardening.md` — обязательные исправления безопасности и надежности для быстрого MVP.
2. `02_action_approval_policy.md` — как убрать многократные подтверждения и дать агенту больше свободы без самоубийственного доверия ко всему подряд.
3. `03_manager_skills_pack.md` — полезные скиллы, сценарии и приемы для управленцев, чтобы демо было про пользу, а не про “бот умеет отвечать”.
4. `04_test_plan.md` — минимальный тест-план, smoke/e2e/security checks и критерии приемки.

## Главный принцип

Сначала делаем систему, которая:

- не передает секреты в LLM prompt;
- не просит пользователя подтвердить одно и то же пять раз;
- умеет выполнять полезные управленческие действия;
- имеет понятные ограничения и тесты;
- быстро запускается через Docker Compose.

Не надо сразу строить корпоративный IAM, multi-tenant Kubernetes и комитет по этике агентов. Но P0-дыры закрываем сразу, иначе это будет не MVP, а красивый способ собрать проблемы в одном контейнере.

## Definition of Done общего уровня

- Все P0 из `01_fast_mvp_hardening.md` реализованы.
- Диалог отправки письма проходит максимум через одно явное подтверждение пользователя.
- Агент имеет набор готовых управленческих команд/скиллов из `03_manager_skills_pack.md`.
- Есть smoke-тесты и минимальные security regression tests из `04_test_plan.md`.
- README и AGENTS.md обновлены под фактическую архитектуру: gateway, dashboard, webapp, Telegram relay, per-user data.

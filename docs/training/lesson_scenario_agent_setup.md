# Lesson Scenario: Настройка персонального агента для управленца

## Цель занятия

Показать слушателям, что агент — это не просто чат с моделью, а настраиваемый рабочий помощник с памятью, файлами, навыками, интеграциями, расписанием и автоматизациями.

После занятия пользователь должен уметь:

- настроить профиль и память агента;
- загрузить файлы и обработать их;
- использовать skills;
- отправить email через approval flow;
- создать напоминание или автоматизацию;
- получить proactive сообщение;
- выполнить поиск и парсинг источников;
- понять границы безопасности и контроля действий.

## Длительность

Рекомендуемо: 90 минут.

Можно сжать до 60 минут, если убрать browsing block или автоматизации. Но тогда это опять будет “посмотрели чатик”, а не занятие про агентов.

## Подготовка преподавателя

До занятия:

- запустить demo instance;
- создать invite-code для группы;
- проверить Telegram bot;
- проверить Files tab;
- проверить email demo account;
- проверить scheduler run-now;
- проверить SearxNG/search tool;
- подготовить 2-3 demo файла:
  - meeting_notes.txt;
  - commercial_offer.md;
  - tasks_messy.txt.

## Структура занятия

### 1. Введение: что такое агент, 10 минут

Ключевая мысль:

- Chatbot отвечает.
- Agent remembers, uses tools, acts with approval, runs scheduled tasks.

Показать архитектуру:

- Web UI;
- Telegram;
- memory;
- files;
- skills;
- tools;
- scheduler;
- approval.

### 2. Регистрация и профиль, 10 минут

Действия слушателя:

1. Открыть webapp.
2. Зарегистрироваться через invite-code.
3. Открыть профиль.
4. Заполнить имя.
5. Настроить краткий SOUL/persona.

Пример persona:

```text
Ты мой деловой AI-помощник. Отвечай кратко, структурно, без воды. Для управленческих задач всегда выделяй: суть, риски, следующие действия.
```

Объяснение:

- SOUL задает устойчивый стиль агента;
- memory хранит факты и предпочтения;
- profile управляет идентичностью агента.

### 3. Память и кросс-сессионный контекст, 15 минут

Действия слушателя:

В чате написать:

```text
Запомни обо мне: я руководитель проекта, предпочитаю короткие письма, не люблю канцелярит, по утрам хочу видеть только главное. У меня аллергия на орехи.
```

Потом:

```text
Что ты обо мне знаешь и как это повлияет на твои ответы?
```

Затем обновить страницу или открыть новый чат и спросить:

```text
Напиши письмо клиенту в моем стиле.
```

Expected result:

- агент вспоминает стиль;
- письмо короткое;
- есть clear next step.

Объяснение преподавателя:

- память не магия;
- память надо редактировать;
- плохая память дает плохого агента;
- личные ограничения вроде аллергий полезны для контекстных задач.

### 4. Файлы: загрузка, папки, обработка, 15 минут

Действия слушателя:

1. Открыть вкладку `Файлы`.
2. Создать папку `Занятие 1`.
3. Загрузить `meeting_notes.txt`.
4. Нажать `Ask agent` или написать:

```text
Проанализируй файл meeting_notes.txt. Выдели задачи, ответственных, сроки и риски.
```

5. Попросить:

```text
Сохрани результат как tasks.md в папку Занятие 1.
```

Expected result:

- файл загружен;
- агент делает таблицу задач;
- результат сохранен и скачивается.

Объяснение:

- files workspace делает агента рабочим инструментом;
- файлы лучше структурировать;
- агент может создавать производные артефакты.

### 5. Skills для управленца, 15 минут

Действия слушателя:

Открыть Skills или использовать готовые prompts.

Skill 1: Meeting Follow-up

```text
Сделай follow-up после встречи: клиент согласился на пилот на 2 недели, мы отправляем КП до пятницы, клиент дает 5 тестовых пользователей, риски — нет доступа к данным и непонятен ответственный с их стороны.
```

Skill 2: Decision Memo

```text
Помоги принять решение: делать сначала Telegram-first MVP или Web-first MVP. Дай варианты, критерии, риски и рекомендацию.
```

Expected result:

- структурированный output;
- задачи;
- next steps;
- можно сохранить в файл;
- можно превратить в письмо.

Объяснение:

- skill — это не “фича модели”, а упакованный рабочий сценарий;
- хороший skill задает формат результата и критерии качества;
- управленческие skills экономят время на повторяемых задачах.

### 6. Email and approval flow, 10 минут

Действия слушателя:

```text
Подготовь письмо самому себе: попроси подключиться к тестированию агента и дать обратную связь до пятницы.
```

Затем:

```text
Отправь это письмо.
```

Expected result:

- агент показывает карточку действия;
- пользователь подтверждает один раз;
- email отправляется;
- нет повторного запроса approval.

Объяснение:

- external actions требуют контроля;
- approval должен быть один, понятный и проверяемый;
- агент не должен самовольно отправлять письма и создавать встречи.

### 7. Шедулинг и проактивность, 15 минут

Действия слушателя:

```text
Напомни мне через 1 минуту проверить ответ клиента.
```

Или:

```text
Каждое утро в 9:00 присылай мне краткий дайджест: важное, просрочки, следующие действия.
```

Для занятия использовать `Run now`, чтобы не ждать утра. Да, удивительно, но учебные занятия редко длятся до следующего дня.

Expected result:

- создается scheduled job;
- пользователь видит его в UI;
- proactive message приходит в Telegram;
- job run появляется в журнале.

Объяснение:

- проактивность отличает агента от чата;
- расписания надо ограничивать;
- recurring jobs должны уважать quota and permissions.

### 8. Поиск и парсинг, 10 минут

Действия слушателя:

```text
Найди 5 источников про AI agents for managers и сделай таблицу: источник, ключевая идея, применимость для руководителя.
```

Затем:

```text
Открой первые 3 источника и сделай краткий research brief.
```

Optional:

```text
Скачай доступные PDF по найденным ссылкам, но не больше 10, и сохрани их в папку Research.
```

Expected result:

- агент ищет через SearxNG;
- парсит страницы;
- сохраняет скачанные файлы;
- делает brief.

Объяснение:

- search != browsing;
- parsing needs limits;
- browser automation нужен только для динамических сайтов;
- Playwright MCP is optional advanced mode.

### 9. Итоговая самостоятельная практика, 10 минут

Задание:

Настройте агента под себя и выполните workflow:

1. Добавьте 3 факта в память.
2. Загрузите файл.
3. Используйте один skill.
4. Создайте письмо с approval.
5. Создайте reminder.
6. Выполните web research.
7. Сохраните результат в файл.

Критерии успеха:

- агент учитывает память;
- output структурирован;
- файл создан или обработан;
- external action прошел через approval;
- reminder/job появился;
- research содержит источники.

## Типовые ошибки слушателей

- Слишком общие команды: “сделай хорошо”. Агент не телепат, хотя иногда делает вид.
- Нет контекста: непонятно для кого, зачем и в каком формате.
- Слишком много задач в одном запросе.
- Не проверяют action card перед подтверждением.
- Ожидают, что агент сам угадает стиль работы.
- Загружают мусорные файлы и ждут executive summary уровня McKinsey.

## Финальный тезис

Персональный агент становится полезным, когда у него есть:

- контекст о пользователе;
- рабочее пространство с файлами;
- набор skills;
- подключенные tools;
- расписание;
- правила подтверждения действий.

Без этого это просто еще один чат. А чатов у человечества уже достаточно, спасибо.

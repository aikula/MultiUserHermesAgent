# Task Extraction

## When to use
Когда есть «сырой» текст (письмо, заметка, протокол, расшифровка) и нужно
вытащить из него задачи в формате, пригодном для трекера.

## Inputs
- Сырой текст.
- Опционально: контекст проекта/команды для уточнения владельцев.

## Output format
- **Таблица задач**: `задача | владелец | срок | приоритет | риск`.
- **Просроченные / без срока** — отдельным блоком, явно помечены.
- **Неявные задачи** (те, что «витают в воздухе» в тексте) — отдельный блок.
- **Предложение по трекеру** — куда завести (Jira, Asana, Notion и т.п. — уточни у юзера).

## Quality checklist
- [ ] Нет задач без владельца (или явно помечены как «нужен владелец»).
- [ ] Срок — дата, не «скоро» / «на неделе».
- [ ] Просрочки определены относительно текущей даты.
- [ ] Приоритет = P0/P1/P2 или эквивалент.
- [ ] Не дублируются с уже понятными задачами.

## Example prompt
Извлеки задачи из письма клиента: «В пятницу ждём фидбек, на следующей неделе
хотели бы обсудить цены на масштабирование, по безопасности у нас вопросы —
можем созвониться?».

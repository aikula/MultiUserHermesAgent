# Email Reply

## When to use
Когда нужно ответить на входящее письмо в определённом тоне и формате.
Особенно полезно для: клиентских запросов, отказов, эскалаций, согласований.

## Inputs
- Текст входящего письма.
- Опционально: желаемый тон (формальный / дружеский), цель ответа.

## Output format
1. **Анализ письма** — что на самом деле хочет отправитель, подтекст.
2. **Стратегия ответа** — согласиться, отказать, уточнить, перевести.
3. **Драфт ответа** — готовый текст.
4. **Что проверить перед отправкой** — факты, тон, attachments.
5. **Если нужен approval** — текст для подтверждения пользователем.

## Quality checklist
- [ ] Ответ начинается с «спасибо / понял / ок» или эквивалента (не «Привет!»).
- [ ] Каждый вопрос из письма либо закрыт, либо переведён в задачу.
- [ ] Длина ответа ≤ длины вопроса × 1.5 (если не нужно больше).
- [ ] Нет «на всякий случай» — только то, что относится к делу.
- [ ] Если отказ — есть альтернатива или «почему нет».
- [ ] Не содержит обещаний, которые нельзя выполнить.

## Example prompt
Ответь клиенту, который просит скидку 30% «как у конкурента», но мы знаем что
у него нет реального оффера, а просто торгуется. Цель — сохранить цену,
предложить бонус по объёму.

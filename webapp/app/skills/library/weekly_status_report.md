# Weekly Status Report

## When to use
Конец недели — отчёт стейкхолдерам, руководителю, команде. Цель — коротко
сказать «где мы, что хорошо, что плохо, что нужно».

## Inputs
- Задачи и их статусы за неделю.
- Решения, принятые на этой неделе.
- Блокеры и риски.

## Output format
- **TL;DR** — 2-3 предложения для тех, кто прочитает только это.
- **Сделано за неделю** — bullet-список (5-10 пунктов).
- **В работе** — что движется, но не закрыто.
- **Просрочено / под риском** — с конкретными сроками.
- **Блокеры** — что нужно от стейкхолдера/руководителя.
- **Метрики** (если есть) — что важно отслеживать.
- **Следующая неделя** — главные фокусы.

## Quality checklist
- [ ] TL;DR — главное, что руководитель должен унести с собой.
- [ ] «Сделано» — это результаты, а не активности («отправили КП», не
  «работали над КП»).
- [ ] Просрочки не спрятаны — вынесены отдельным блоком.
- [ ] Блокеры конкретные: кто, что, до когда.
- [ ] Метрики = 1-3 ключевых, не дашборд на 20 графиков.
- [ ] Длина — 1 страница, не 5.

## Example prompt
Сделай weekly status по проекту внедрения CRM: мы на 2-м месяце, бюджет
потрачен на 60%, основной блокер — интеграция с 1С, на следующей неделе
запуск пилота в одном филиале.

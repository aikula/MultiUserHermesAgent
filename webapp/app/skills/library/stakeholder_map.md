# Stakeholder Map

## When to use
Когда проект/инициатива затрагивает несколько людей/отделов и нужно понять,
кто реально влияет на результат, а кто только «в курсе».

## Inputs
- Проект / инициатива.
- Список людей и отделов, которых это касается.

## Output format
- **Карта по 4 квадрантам**:
  - **High power, high interest** — управлять внимательно (key players).
  - **High power, low interest** — держать удовлетворёнными.
  - **Low power, high interest** — информировать.
  - **Low power, low interest** — мониторить.
- **Для каждого стейкхолдера**: роль, интерес, что от него нужно, как часто
  коммуницировать.
- **Скрытые стейкхолдеры** — кого забыли упомянуть (подумай: безопасность,
  юристы, финансы, саппорт).
- **План первого касания** — кому написать сегодня / на этой неделе.

## Quality checklist
- [ ] Все четыре квадранта заполнены, не пустые.
- [ ] Скрытые стейкхолдеры названы явно.
- [ ] Для каждого — конкретный «следующий шаг», не «поддерживать отношения».
- [ ] Учтён формальный и неформальный influence.
- [ ] Указано, кто может заблокировать, а кто — ускорить.

## Example prompt
Сделай stakeholder map для запуска нового продукта внутри банка:
задействованы IT, риски, комплаенс, продажи, операционка, и топ-менеджмент.
Я — продакт-менеджер.

# Risk Review

## When to use
Перед запуском, инвестицией, важным обязательством, а также на ретроспективе
проекта. Цель — заранее увидеть то, что пойдёт не так.

## Inputs
- Описание инициативы / проекта / обязательства.
- Опционально: предыдущие ретроспективы, известные проблемы.

## Output format
- **Категоризированные риски** — по группам:
  - **Люди** (зависим от одного человека, нет преемника).
  - **Технология** (не проверено, vendor lock-in).
  - **Бизнес** (рынок, регуляторика, конкуренты).
  - **Финансы** (cash flow, девальвация, перерасход).
  - **Коммуникации** (стейкхолдер не в курсе, ожидания неверные).
- **Для каждого риска**: вероятность (низкая/средняя/высокая), импакт, mitigation.
- **Top-3 риска** — на что смотреть в первую очередь.
- **Сигналы раннего предупреждения** — что должно насторожить за 2-4 недели до проблемы.

## Quality checklist
- [ ] Рисков минимум 5, лучше 8-12.
- [ ] Каждый риск имеет mitigation, не только описание.
- [ ] Top-3 — действительно топ, а не «всё важно».
- [ ] Есть хотя бы один риск по людям (обычно самый недооценённый).
- [ ] Указаны сигналы, по которым можно отслеживать риск в динамике.

## Example prompt
Сделай risk review для запуска B2B SaaS за 3 месяца: команда 4 человека,
бюджет 2 млн руб., первый клиент уже подписан на пилот, но не платит.

# Decision Memo

## When to use
Когда нужно принять или зафиксировать решение (MVP-стратегия, выбор
подрядчика, приоритизация фич, смена процесса). Используй перед тем, как
объявлять решение коллегам или стейкхолдерам.

## Inputs
- Вопрос / развилка.
- Опционально: ограничения (бюджет, время, ресурсы), стейкхолдеры.

## Output format
1. **Контекст** — почему решение нужно сейчас.
2. **Варианты** — 2-4, каждый с плюсами, минусами, рисками.
3. **Критерии выбора** — что для нас важнее всего.
4. **Рекомендация** — какой вариант и почему.
5. **Что нужно подтвердить** — у кого, до какого срока.
6. **Что произойдёт, если не решать** — статус-кво и его цена.

## Quality checklist
- [ ] Рассмотрено минимум 2 варианта (иначе это не развилка).
- [ ] Для каждого варианта: цена, риски, что придётся отказать.
- [ ] Рекомендация одна, не «зависит от контекста».
- [ ] Указано, чьё одобрение нужно.
- [ ] Обозначен дедлайн принятия.

## Example prompt
Помоги принять решение: делать сначала Telegram-first MVP или Web-first MVP.
Дай варианты, критерии, риски и рекомендацию.
